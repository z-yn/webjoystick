UIManager

Help to create some components with a default action.

new UIManager(gamepad)

Create a UIManager for a gamepad, a UIManager can help to create some components with a default action.
Parameters:
Name Type Description

init()

This init method is used for create action for buttons, sticks, and sliders.
Also this method can help to prevent some default events, for example, slide to forward or back in UCBrowser.

initButton()

This method is used for creating buttons according to divs' className, a div with class name "pad-button" will automatically send message when it is pressed or released.
Message is in two types:"buttonDown", "buttonUp".
The gamedata for both "buttonDown" and "buttonUp" messages is defined as:
{"id": button.id}

initSlider()

This method is used for creating sliders according to divs, a div with class name "slider" will be append a handle as a child, and automatically send message when the handle is dragged.
Message is in three types:"sliderStart", "sliderMove", "sliderEnd".
The gamedata of a message is defined as:
For "sliderStart" and "sliderMove":{"id": slider.id, "position": x}
For "sliderEnd":{id: slider.id};
And position above is normalized in range(-1, 1);

initStick()

This method is used for creating sticks according to divs' class name, a div with class name "stick" will be append a handle as a child, and automatically send message when the handle is dragged.
Message is in three types:"dragStart", "dragMove", "dragEnd".
The gamedata of a message is defined as:
For "dragStart" and "dragMove":{"id": stick.id, "position":{"x": x, "y": y}}
For "dragEnd":{id: stick.id};
And position.x and position.y above is normalized in range(-1, 1);
