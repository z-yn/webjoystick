Class: Gamepad

Gamepad

The wrapper in the gamepad, exchanges message with station.

new Gamepad(url, uid)

To construct a gamepad object on the gamepad, which can call the device API and send event data to the game, the gamepad won't connect to the WebSocket by default, please call Gamepad.connect() before use.
Parameters:
Name Type Description
url The WebSocket url to connect to.
uid The id of the gamepad, in multi player games, this will help to distinct events generated from different gamepads.

devicemotionEvt(event)

Method to check if the device is shaked and send the "devicemotion" event to the station.
The gamedata of the shake message is defined as:
{"speed": speed} and the message type is named as "shake".
The gamedata of the acceleration event is defined as:
{"x": x, "y": y, "z":z}
And the definition is same as the HTML5 standard.
The speed is the delta of the acceleration on the x axis, so just hold the gamepad in landscape view and shake the device left or right.
Parameters:
Name Type Description
event The device motion event.

deviceorientationEvt(data)

Method to handle the orientation event, this method is called on both "devicemotion" and "deviceorientation", for a device without "deviceorientation", we use the accelerationIncludingGravity to determinate the orientation.
The gamedata of this message is defined as:
{"alpha": alpha, "beta": beta, "gamma": gamma}, and message type is named as "deviceorientation".
The alpha, beta, gamma above is finally converted -180~180, same as the deviceorientation event in the definition in the HTML5 standard.
Parameters:
Name Type Description
data Maybe orientation, or motion, so we used many "if"s.
