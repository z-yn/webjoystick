Foo-Library

We hide the name of our library because we are participating a competition.
This library is a framework for creating multi-device games via WebSocket connection.

A Model of Games Developed under This Library

Devices

In the game, device are used as a "station" or a "gamepad".
Station refers to a device with big screen and high performance, and runs the main scene of a game.
Gamepad refers to lightweight devices without high performance, and are used as controllers of a game.

Network

The game should run among a low-delay network, for examples, a computer and some cellphones connected to one wireless router, or some cellphones connected to a ad-hoc network created by a computer or pad.
So a station with ad-hoc support can run the game anywhere.
As for online games, because the Wi-Fi adapter is working in the ad-hoc network, is not easy to be taken anywhere.

Browser

Browsers must support HTML5 and WebSocket, also, WebGL support on the station can greatly improve the game experience.
Browsers on gamepads would better has rich device API support like device orientation, device motion, and vibrate feedback.
We suggest to use chrome browser on the station, and UCBrowser on the gamepad.

Communication

A WebSocket server should run on the station, and the foreground of the game can run on the browser of the station, and gamepads should connect to the WebSocket and do data exchanges with station.

Expected Effect

A high-performance station can run the main scene of a game, brings great graphics, and gamepads with device orientation, device motion, and vibrate feedback can become a gamepad no worse than a gamepad from PS3.
So we can achieve very good game experience.

Using This Library

Refer JSs and CSSs

On the station, you should refer to "gamewatcher.js", and use the "on" function to register listeners to messages from gamepads.
On the gamepads, you should refer to "gamepad.js", and so you can send data from device API or screen event to the station, also, we have encapsulate the device API, and automatically send data from device API.
And there we have encapsulated some basic controller components, for example sticks, sliders, and buttons, and you should refer to "UIManager.js" and "components.css".
